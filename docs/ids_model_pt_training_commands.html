IDS Training and Final model.pt Evaluation Commands

Every card contains one complete command chain. Click its button to copy the entire chain.

Important: Run only one training command at a time. The repository training lock rejects overlapping trainers. The dedicated PenDHRL comparison below evaluates both model.pt and model_best.pt; the architecture-specific train-and-evaluate commands use final model.pt.
200-epoch IDS curriculum:
  • Epochs 0–59: baseline with IDS disabled
  • Epochs 60–119: medium IDS
  • Epochs 120–199: full IDS
  • Final main.py --eval: full IDS difficulty

PenDHRL baseline: fresh start, interruption recovery, and comparison

P1. Start fresh PenDHRL baseline training

Starts a new 200-epoch, three-scenario PenDHRL baseline with the IDS curriculum. It contains no LLM flags. CPUs 16–31 are the verified training CPU set.

cd /home/user5/NASimEmu-enhanced/NASimEmu-agents && \
source ../venv/bin/activate && \
set -euo pipefail && \
PYTHON=../venv/bin/python && \

taskset -c "$CPUSET" "$PYTHON" main.py \
  "$SCENARIOS" \

P3. Evaluate current model.pt and model_best.pt

Takes stable snapshots from run 6892pgk3, then evaluates the two checkpoints sequentially on the verified stable CPUs 16–31. --eval automatically selects full IDS difficulty. Results are saved as separate logs.

cd /home/user5/NASimEmu-enhanced/NASimEmu-agents && \
source ../venv/bin/activate && \
set -euo pipefail && \
PYTHON=../venv/bin/python && \
CPUSET=16-31 && \
RUN_DIR="$PWD/wandb/offline-run-20260717_115521-6892pgk3" && \
SCENARIOS="../scenarios/corp_100hosts_dynamic.v2.yaml:../scenarios/corp_100hosts_dynamic_varA.v2.yaml:../scenarios/corp_100hosts_dynamic_varB.v2.yaml" && \
TEST_SCENARIO="../scenarios/corp_100hosts_dynamic.v2.yaml" && \
STAMP="$(date +%Y%m%d_%H%M%S)" && \
SNAPSHOT_DIR="$PWD/training_data/eval_snapshots/pendhrl_6892pgk3_${STAMP}" && \
RESULT_DIR="$PWD/training_data/eval_results/pendhrl_6892pgk3_${STAMP}" && \
mkdir -p "$SNAPSHOT_DIR" "$RESULT_DIR" /tmp/nasimemu-matplotlib /tmp/nasimemu-wandb-cache && \
test -s "$RUN_DIR/files/model.pt" && \
test -s "$RUN_DIR/files/model_best.pt" && \
cp "$RUN_DIR/files/model.pt" "$SNAPSHOT_DIR/model.pt" && \
cp "$RUN_DIR/files/model_best.pt" "$SNAPSHOT_DIR/model_best.pt" && \
test -s "$SNAPSHOT_DIR/model.pt" && \
test -s "$SNAPSHOT_DIR/model_best.pt" && \
export OPENBLAS_NUM_THREADS=1 OMP_NUM_THREADS=1 MKL_NUM_THREADS=1 NUMEXPR_NUM_THREADS=1 MPLCONFIGDIR=/tmp/nasimemu-matplotlib WANDB_CACHE_DIR=/tmp/nasimemu-wandb-cache WANDB_MODE=offline WANDB_DIR="$PWD" && \
for CHECKPOINT_NAME in model.pt model_best.pt; do \
  CHECKPOINT="$SNAPSHOT_DIR/$CHECKPOINT_NAME"; \
  LOG_NAME="${CHECKPOINT_NAME%.pt}.log"; \
  echo "=== FULL-IDS DYNAMIC EVALUATION: $CHECKPOINT_NAME ==="; \
  taskset -c "$CPUSET" "$PYTHON" main.py \
    "$SCENARIOS" \
    --test_scenario "$TEST_SCENARIO" \
    --eval \
    -load_model "$CHECKPOINT" \
    -device cpu -cpus 8 -batch 16 \
    -net_class NASimNetDHRL \
    -use_a_t \
    -episode_step_limit 400 \
    -observation_format graph_v2 \
    -seed 1 2>&1 | tee "$RESULT_DIR/$LOG_NAME"; \
done && \
echo "=== BOTH EVALUATIONS COMPLETE ===" && \
echo "Snapshots: $SNAPSHOT_DIR" && \
echo "Results:   $RESULT_DIR"

5. Live LLM DAgger: train and evaluate final-round model.pt

Requires Ollama running locally and the qwen2.5:3b-instruct model already pulled. Performs three DAgger rounds and then standard full-IDS evaluation.

cd /home/user5/NASimEmu-enhanced/NASimEmu-agents && \
source ../venv/bin/activate && \
PYTHON=../venv/bin/python && \
CPUSET=16-31 && \
SCENARIOS="../scenarios/corp_100hosts_dynamic.v2.yaml:../scenarios/corp_100hosts_dynamic_varA.v2.yaml:../scenarios/corp_100hosts_dynamic_varB.v2.yaml" && \
TEST_SCENARIO="../scenarios/corp_100hosts_dynamic.v2.yaml" && \
export OPENBLAS_NUM_THREADS=1 OMP_NUM_THREADS=1 MKL_NUM_THREADS=1 NUMEXPR_NUM_THREADS=1 MPLCONFIGDIR=/tmp/nasimemu-matplotlib WANDB_CACHE_DIR=/tmp/nasimemu-wandb-cache WANDB_MODE=offline WANDB_DIR="$PWD" && \

command -v ollama >/dev/null && \

ollama list | grep -Fq 'qwen2.5:3b-instruct' && \
taskset -c "$CPUSET" "$PYTHON" experiments/dagger_loop.py \
  --rounds 3 \
  --records_per_round 700 \
  --scenario "$SCENARIOS" \
  --teacher_backend llm \
  --model qwen2.5:3b-instruct \
  --out_dir training_data/llm_teacher_dataset \
  --checkpoint_dir training_data/dagger_checkpoints \
  --train_args "$SCENARIOS --test_scenario $TEST_SCENARIO -device cpu -cpus 8 -batch 128 -epoch 100 -max_epochs 200 --no_debug -net_class NASimNetDHRL -force_continue_epochs 0 -use_a_t -episode_step_limit 400 -observation_format graph_v2 -lr 0.0007 -alpha_h 0.02 --sched_lr_rate 10000 --sched_lr_factor 0.8 --sched_lr_min 0.0003 --sched_alpha_h_rate 15000 --sched_alpha_h_factor 0.5 --sched_alpha_h_min 0.005 -seed 1 --llm_distill --llm_distill_warmup_epochs 3 --llm_distill_rl_only_frac 0.1" && \
LIVE_RUN_DIR="$(readlink -f wandb/latest-run)" && \
LIVE_CHECKPOINT="$LIVE_RUN_DIR/files/model.pt" && \
test -s "$LIVE_CHECKPOINT" && \
